# API通用描述

<Callout type="info">
本系统主要为商户提供虚拟币支付功能，所有的请求参数通过加密提交，支持多种币种和链条，并提供风险评分机制。
</Callout>

---

## API格式约定

<Badge type="info">加密说明</Badge>  
本文档中描述的所有请求参数都是组装成 JSON 格式的请求体，使用 **RSA 公钥** 对整个 JSON 请求体进行加密，将加密后的数据进行 **Base64 编码**，并将 Base64 编码后的数据作为 `data` 参数提交给 API 接口。

## 钱包地址格式约定

### **格式要求**

- **TRON** 的钱包地址中字母 **区分大小写**。
- **ETH** 的钱包地址中字母统一使用 **小写**。

## 风险评分参考

<Badge type="warning">风险等级</Badge>

当前系统默认的风险分数等级如下：

- 🟢 **低风险**：`0 ~ 30`，无风险或低风险。
- 🟡 **中风险**：`31 ~ 64`，可能跟中高风险地址有过交易。
- 🟠 **高风险**：`65 ~ 84`，跟中高风险的地址有过交易，建议做隔离。
- 🔴 **严重风险**：`85 ~ 100`，跟高危风险的地址有过交易，建议立即做风险隔离。
